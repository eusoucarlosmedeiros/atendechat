## Story
**As a** integração typebot
**I want** que `typebotListener.ts` envie respostas e presença via uazapi
**So that** os bots configurados continuem respondendo após o cutover.

## Acceptance Criteria
1. Em `services/TypebotServices/typebotListener.ts`:
   - `wbot.sendMessage(jid, { text })` → `SendText(whatsapp, { number, text })`.
   - `wbot.sendMessage(jid, { image: { url } })` → `SendMedia(whatsapp, { number, type:"image", file: url })`.
   - `wbot.sendMessage(jid, { audio: { url, ptt: true } })` → `SendMedia(whatsapp, { number, type:"ptt", file: url })`.
   - `wbot.presenceSubscribe(jid)` + `wbot.sendPresenceUpdate("composing", jid)` → `SendPresence(whatsapp, { number, type:"composing" })` (criar wrapper se ainda não existe).
   - `wbot.sendPresenceUpdate("paused", jid)` → `SendPresence(whatsapp, { number, type:"paused" })`.
2. Delay/timing dos bots preservado (`typebotDelayMessage` continua funcionando).
3. Build limpo + tests.
4. **Smoke test**: configurar 1 typebot simples com 2 etapas (texto+pergunta), receber resposta e validar que typebot continua respondendo.

## Dev Notes

### Cuidado especial — timing
O typebot manda `composing`, espera `delay`, manda mensagem, manda `paused`. Latência da uazapi (HTTP) é maior que Baileys (WebSocket). Validar se o usuário ainda vê o "Digitando..." antes da mensagem. Se for percebido, encurtar `delay` ou enviar presence + mensagem em paralelo.

### Wrapper `SendPresence` (se faltar criar em 01.02)
```ts
// services/UazapiServices/chat/SendPresence.ts
export interface SendPresenceParams {
  number: string;
  type: "composing" | "recording" | "paused";
  duration?: number; // ms, max 300000
}
const SendPresence = async (whatsapp, params) => {
  const client = getUazapiClient(whatsapp);
  const res = await client.post("/message/presence", params);
  return res.data;
};
```
