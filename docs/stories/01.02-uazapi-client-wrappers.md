# Story 01.02 — Cliente uazapi + wrappers tipados

## Status
**Draft**

## Story
**As a** desenvolvedor backend
**I want** uma camada cliente HTTP única para a uazapi (`libs/uazapi.ts` + `services/UazapiServices/*`) com wrappers tipados de cada endpoint usado
**So that** as stories de webhook, send e session usem uma API interna consistente em vez de espalhar `axios.post(...)` pelo código.

## Acceptance Criteria
1. `backend/src/libs/uazapi.ts` exporta uma função `getUazapiClient(whatsapp: Whatsapp)` que retorna uma instância axios configurada com:
   - `baseURL` = `whatsapp.uazapiBaseUrl || process.env.UAZAPI_BASE_URL`
   - Header `token: whatsapp.uazapiToken` (auto-injetado em todas as requests).
   - Timeout default de 30s (configurável via env `UAZAPI_TIMEOUT_MS`).
   - Retry automático em 5xx e network errors (3 tentativas, backoff exponencial 1s/2s/4s) — usar `axios-retry` ou implementação manual.
   - Logger pino: log de cada request/response em level `debug`, erros em `warn`.
2. `backend/src/libs/uazapiAdmin.ts` (separado, sem dependência de Whatsapp): expõe instância axios para chamadas com `admintoken` (criar instâncias).
3. `backend/src/services/UazapiServices/instance/`:
   - `InitInstance.ts` — `POST /instance/init` → retorna `{ id, token }`
   - `ConnectInstance.ts` — `POST /instance/connect` → retorna `{ qrcode? }`
   - `DisconnectInstance.ts` — `POST /instance/disconnect`
   - `GetInstanceStatus.ts` — `GET /instance/status`
4. `backend/src/services/UazapiServices/send/`:
   - `SendText.ts` — `POST /send/text` (suporta `replyid`, `mentions`, `linkPreview`)
   - `SendMedia.ts` — `POST /send/media` (image/video/audio/ptt/document/sticker)
   - `SendLocation.ts` — `POST /send/location`
   - `SendContact.ts` — `POST /send/contact`
   - `SendReact.ts` — `POST /message/react`
5. `backend/src/services/UazapiServices/chat/`:
   - `CheckNumber.ts` — `POST /chat/check`
   - `ReadChat.ts` — `POST /chat/read`
   - `DeleteMessage.ts` — `POST /message/delete`
   - `EditMessage.ts` — `POST /message/edit`
   - `ChatDetails.ts` — `POST /chat/details`
6. `backend/src/services/UazapiServices/group/`:
   - `GetGroupInfo.ts` — `POST /group/info`
   - `ListGroups.ts` — `GET /group/list`
   - (CRUD básico — adicionar/remover participantes pode ficar para Story 01.06 se não usado em prod)
7. Cada wrapper define **interfaces TypeScript** para request body e response (sem `any`).
8. Erros 4xx/5xx são propagados como `AppError("ERR_UAZAPI_*", code)` com mensagem amigável + `originalError` em metadados.
9. **HTTP 429 (rate-limited)** NÃO faz retry imediato: aplica backoff exponencial com jitter, máximo 3 tentativas, delays 5s/15s/45s. Após esgotar, propaga `ERR_UAZAPI_RATE_LIMITED` para o caller decidir (fila Bull pode reenfileirar).
10. **Sem chamada real à uazapi nos testes** — usar nock ou jest mock para HTTP.
11. `npx tsc --noEmit` passa limpo.

## Dev Notes

### Estrutura sugerida do cliente
```ts
// libs/uazapi.ts
import axios, { AxiosInstance } from "axios";
import axiosRetry from "axios-retry";
import Whatsapp from "../models/Whatsapp";
import { logger } from "../utils/logger";

export const getUazapiClient = (whatsapp: Whatsapp): AxiosInstance => {
  const baseURL = whatsapp.uazapiBaseUrl || process.env.UAZAPI_BASE_URL;
  if (!baseURL) throw new AppError("UAZAPI base URL not configured");
  if (!whatsapp.uazapiToken) throw new AppError("Whatsapp instance not initialized");

  const client = axios.create({
    baseURL,
    timeout: Number(process.env.UAZAPI_TIMEOUT_MS || 30000),
    headers: { token: whatsapp.uazapiToken, "Content-Type": "application/json" }
  });

  axiosRetry(client, {
    retries: 3,
    retryDelay: axiosRetry.exponentialDelay,
    retryCondition: (err) => axiosRetry.isNetworkOrIdempotentRequestError(err)
                          || (err.response?.status >= 500)
  });

  client.interceptors.request.use(req => {
    logger.debug(`[uazapi] ${req.method?.toUpperCase()} ${req.url}`);
    return req;
  });
  client.interceptors.response.use(
    res => res,
    err => {
      logger.warn(`[uazapi] ${err.config?.url} → ${err.response?.status}: ${err.message}`);
      throw err;
    }
  );

  return client;
};
```

### Convenção dos wrappers
Cada wrapper recebe `Whatsapp` (não `whatsappId`) para evitar 1 query extra por chamada. O caller resolve a instância antes:

```ts
// services/UazapiServices/send/SendText.ts
export interface SendTextParams {
  number: string;        // E.164 ou JID
  text: string;
  replyid?: string;
  mentions?: string;     // CSV
  linkPreview?: boolean;
  delay?: number;
}
export interface SendTextResponse {
  id: string;
  status: string;
  timestamp: number;
}
const SendText = async (whatsapp: Whatsapp, params: SendTextParams): Promise<SendTextResponse> => {
  const client = getUazapiClient(whatsapp);
  const res = await client.post("/send/text", params);
  return res.data;
};
export default SendText;
```

### Mapping de erros uazapi → AppError
| Status uazapi | Causa | AppError |
|--------------|-------|----------|
| 400 | payload inválido / número malformatado | `ERR_UAZAPI_BAD_REQUEST` |
| 401 | token inválido / sem sessão | `ERR_UAZAPI_UNAUTHORIZED` |
| 404 | instância/grupo/msg não encontrado | `ERR_UAZAPI_NOT_FOUND` |
| 429 | rate limit | `ERR_UAZAPI_RATE_LIMITED` |
| 5xx | erro interno uazapi | `ERR_UAZAPI_SERVER_ERROR` |
