# Story 01.06c — Validação de número, foto, importação de contatos

## Status
**Draft** — depende de 01.02 Done.

## Story
**As a** sistema Atendechat
**I want** validar números (CheckNumber), pegar foto de perfil (GetProfilePicUrl) e importar contatos via uazapi
**So that** essas funções utilitárias continuem operacionais após cutover.

## Acceptance Criteria
1. `services/WbotServices/CheckNumber.ts`: `wbot.onWhatsApp(jid)` → `CheckNumber(whatsapp, { number })` (`POST /chat/check`). Retorna `{ exists: boolean, jid?: string }`.
2. `services/WbotServices/CheckIsValidContact.ts`: idem, usa CheckNumber.
3. `services/WbotServices/GetProfilePicUrl.ts`: `wbot.profilePictureUrl(jid)` → `ChatDetails(whatsapp, { number }).profilePicUrl` (`POST /chat/details`).
4. `services/WbotServices/ImportContactsService.ts`: `wbot.contacts` (sync iterator) → `ListContacts(whatsapp)` (`GET /contacts`) com paginação.
5. **Importação grande** (1000+ contatos) usa paginação e respeita rate limit. Em caso de 429 propagado, salvar progresso e re-enfileirar.
6. Build limpo + tests.
7. **Smoke**: validar 1 número conhecido (deve retornar exists=true), validar 1 número inexistente (false), pegar foto de 1 contato.

## Dev Notes

### Comportamento de erro em CheckNumber
Antes (Baileys): se número não existe, retorna array vazio.
Depois (uazapi): `POST /chat/check` retorna `{ exists: false, ... }` ou `{ exists: true, jid }`.

Adaptar contrato dos callers: passar a checar `result.exists` em vez de `result.length > 0`.

### Paginação em ListContacts
A uazapi expõe `POST /contacts/list` com `{ page, pageSize, offset }`. Iterar até resposta retornar lista vazia. Sugiro `pageSize: 100` por padrão.
