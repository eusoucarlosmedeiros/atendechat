1. Pacote `baileys` removido do `package.json` e `package-lock.json`.
2. Pacote `@hapi/boom` removido se não for mais usado por nenhum import (verificar antes — outros lugares podem usar para AppError).
3. Arquivos deletados:
   - `backend/src/libs/wbot.ts`
   - `backend/src/libs/store.d.ts`
   - `backend/src/helpers/authState.ts`
   - `backend/src/helpers/LidPnResolver.ts` (lógica não é mais necessária — uazapi resolve LID/PN sozinha)
   - `backend/src/helpers/GetWbotMessage.ts`
   - `backend/src/services/WbotServices/wbotMessageListener.ts` (lógica migrou pra `UazapiWebhookServices/handleMessages.ts`)
   - `backend/src/services/WbotServices/wbotMonitor.ts` (calls migraram pra `handleCall`)
   - `backend/src/services/WbotServices/providers.ts` (era util Baileys-specific)
   - `backend/src/services/BaileysServices/CreateOrUpdateBaileysService.ts`
   - `backend/src/services/BaileysChatServices/CreateOrUpdateBaileysChatService.ts`

4. Ajustar `helpers/GetTicketWbot.ts` — retornar `Whatsapp` model (já alterado em 01.05) e/ou renomear para `GetTicketWhatsapp.ts`.
5. Renomear `services/WbotServices/` para `services/WhatsappServices/` (opcional, mas melhora clareza). **Decisão:** **manter o nome** WbotServices para evitar churn — só revisar conteúdo.
6. Variáveis de ambiente novas adicionadas em `.env.example`:
   ```
   UAZAPI_BASE_URL=https://api.uazapi.com
   UAZAPI_ADMIN_TOKEN=
   UAZAPI_WEBHOOK_BASE_URL=https://api.atendechat.example.com

   ```
7. `npx tsc --noEmit` passa limpo.
8. `npm run build` produz `dist/` sem erros.
9. Smoke test em produção (após deploy):
   - Reconectar todas as instâncias (re-QR).
   - Receber 1 mensagem real.
   - Enviar 1 mensagem real.
   - Validar campanhas.
10. Documentação atualizada em `docs/architecture/ADR-001-baileys-to-uazapi.md` marcando status como "Implemented".

## Dev Notes

### Plano de comunicação ao cliente (template)
```
Olá! Vamos atualizar o sistema do Atendechat na noite de [DATA].
Após a atualização, será necessário reconectar o WhatsApp escaneando
o QR code novamente. Tempo estimado de indisponibilidade: 30 minutos.
```

### Ordem cirúrgica do PR
1. PR-01: Stories 01.01-01.06 + flag de feature `WHATSAPP_PROVIDER` (`baileys|uazapi`) → não, decisão #2 do ADR foi **cutover único**, sem flag.
2. PR-02 (este): cutover real.
   - Branch `cutover/baileys-uazapi`.
   - Code review obrigatório.
   - Deploy planejado em janela de baixa atividade.

### Rollback
Em caso de quebra grave em produção:
1. `pm2 stop atendechat-backend`
2. `git revert <merge-commit>` na branch main + `git push`
3. `update.sh` reverte código.
4. As instâncias continuarão **desconectadas** (sessões Baileys foram apagadas) — clientes precisam re-QR Baileys.
5. Comunicar incidente.

⚠️ **O rollback não é trivial — Baileys não tem state algum no servidor uazapi.** Por isso o smoke test pré-cutover é crítico.
