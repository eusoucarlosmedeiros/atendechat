# Story 01.05 — Sessão / QR / Status via uazapi

## Status
**Draft** — depende de **01.02 Done** (cliente uazapi).

## Story
**As a** administrador do Atendechat
**I want** abrir uma nova conexão WhatsApp pelo painel, escanear QR e ver o status mudar para CONNECTED em tempo real
**So that** novas instâncias possam ser configuradas sem mexer em código.

## Acceptance Criteria
1. `StartWhatsAppSession.ts`:
   - Se `whatsapp.uazapiInstanceId` é null: chama `InitInstance` (admin) → grava `uazapiInstanceId`, `uazapiToken`, `uazapiBaseUrl`.
   - Gera `uazapiWebhookSecret` (16 bytes random hex).
   - Configura webhook na uazapi: `POST /webhook` com URL `${UAZAPI_WEBHOOK_BASE_URL}/uazapi/webhook/${secret}` e events `["messages","messages_update","connection","call","contacts","presence"]` + `excludeMessages: ["wasSentByApi"]`.
   - Chama `ConnectInstance` → recebe QR base64 → grava em `Whatsapps.qrcode`, status `qrcode`.
   - Emite socket.io `whatsappSession` action `update` (mantendo retrocompat com frontend).
2. `WhatsAppSessionController.ts`:
   - `store` → `StartWhatsAppSession`.
   - `update` → `DisconnectInstance` (se conectado) + `StartWhatsAppSession`.
   - `remove` → `DisconnectInstance` + `whatsapp.update({ status: "DISCONNECTED" })`.
3. `WhatsAppController.ts` `index`/`show`:
   - Status pode ser refrescado opcionalmente via `GetInstanceStatus` na chamada (cuidado com latência — fazer em background se necessário).
4. `StartAllWhatsAppsSessions.ts` no boot do servidor: itera `Whatsapps` com `status != DISCONNECTED` e tenta reconectar (re-config webhook idempotente — se já existir, skip).
5. Reconexão automática em caso de `disconnect` evento webhook (já tratado em `handleConnection`).
6. Frontend continua funcionando **sem mudanças** (recebe QR via socket.io igual hoje).
7. **Tratamento de falha parcial em `StartWhatsAppSession`**: se `InitInstance` sucede mas configuração de webhook ou `ConnectInstance` falha, fazer rollback chamando `DisconnectInstance` e marcando `Whatsapps.status = DISCONNECTED + uazapiInstanceId = NULL`. Estado nunca pode ficar "instância criada mas webhook ausente" — isso causaria mensagens chegando sem nosso backend escutar.
8. Tests + build limpo.

## Dev Notes

### Pseudocódigo de `StartWhatsAppSession`
```ts
import crypto from "crypto";
import InitInstance from "../UazapiServices/instance/InitInstance";
import ConnectInstance from "../UazapiServices/instance/ConnectInstance";
import { adminClient } from "../../libs/uazapiAdmin";

export const StartWhatsAppSession = async (whatsapp: Whatsapp, companyId: number) => {
  const io = getIO();

  // 1. Garantir instância criada na uazapi
  if (!whatsapp.uazapiInstanceId) {
    const init = await InitInstance({ name: whatsapp.name });  // usa admintoken
    await whatsapp.update({
      uazapiInstanceId: init.id,
      uazapiToken: init.token,
      uazapiBaseUrl: process.env.UAZAPI_BASE_URL,
      uazapiWebhookSecret: crypto.randomBytes(16).toString("hex")
    });
    await whatsapp.reload();
  }

  // 2. Configurar webhook (idempotente — se já existe, atualiza)
  const webhookUrl = `${process.env.UAZAPI_WEBHOOK_BASE_URL}/uazapi/webhook/${whatsapp.uazapiWebhookSecret}`;
  await ConfigureWebhook(whatsapp, {
    url: webhookUrl,
    events: ["messages","messages_update","connection","call","contacts","presence"],
    excludeMessages: ["wasSentByApi"],
    enabled: true
  });

  // 3. Conectar (gera QR ou usa sessão existente)
  const connectRes = await ConnectInstance(whatsapp);
  if (connectRes.qrcode) {
    await whatsapp.update({ qrcode: connectRes.qrcode, status: "qrcode" });
    io.to(`company-${companyId}-mainchannel`).emit(
      `company-${companyId}-whatsappSession`,
      { action: "update", session: whatsapp }
    );
  } else if (connectRes.connected) {
    await whatsapp.update({ status: "CONNECTED", qrcode: "" });
    io.to(`company-${companyId}-mainchannel`).emit(
      `company-${companyId}-whatsappSession`,
      { action: "update", session: whatsapp }
    );
  }
};
```

### Sobre `helpers/GetTicketWbot.ts`
Hoje retorna o `Session` (cliente Baileys). Após a migração, **não há mais "client"** — só o model `Whatsapp` com config uazapi. Refatorar para retornar `Whatsapp`:

```ts
// helpers/GetTicketWbot.ts (refatorado)
const GetTicketWbot = async (ticket: Ticket): Promise<Whatsapp> => {
  if (!ticket.whatsappId) {
    const defaultWhatsapp = await GetDefaultWhatsApp(ticket.user.id);
    await ticket.$set("whatsapp", defaultWhatsapp);
  }
  return ShowWhatsAppService(ticket.whatsappId, ticket.companyId);
};
```
